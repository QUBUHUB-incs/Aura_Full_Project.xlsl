# XLSL Format

XLSL (eXtended Logic & Science Language) is Aura’s core computation format.  
It blends scripting, mathematics, and structured pipelines into a single reproducible file.

## Why XLSL?
- Human-readable  
- Strictly deterministic  
- Reproducible computation  
- Perfect for AI, STEM, simulation, and quantum workflows  

---

# Basic Structure

```xlsl
task ComputeEnergy {
    input mass: float
    constant c = 299792458

    compute {
        return mass * c * c
    }
}

```
Each XLSL file can define:
	•	task → A runnable unit
	•	dataset → Load/transform data
	•	model → Train/evaluate ML models
	•	pipeline → Chain tasks

⸻

    ```mdx
Running a Task

aura run compute-energy --mass 0.5

Output:

4.493775893e16 joules

⸻
    
    ```xlsl
Pipelines Example

pipeline ResearchFlow {
    step load -> preprocess -> train -> evaluate
}

Aura maps it into a full dependency graph.
```
⸻
    ```cpp
XLSL + AI Mode

ai Reason {
    prompt """
    Summarize the dataset and propose a research hypothesis.
    """
}
```
Aura’s runtime handles context, caching, and reproducibility.

⸻

# NIX Format

NIX powers Aura’s reproducible environments.  
Think Dockerfiles — but scientific, simple, cross-platform, and integrated.
```
## Example

    ```nix
env PhysicsLab {
    python = "3.10"
    pip = [
        "numpy"
        "scipy"
        "matplotlib"
    ]

    system = {
        cuda = true
        memory = "8GB"
    }
}

Run:

aura env build physics-lab

```
⸻

Features
	•	Full dependency locking
	•	CPU/GPU awareness
	•	Cross-platform consistency
	•	One file = entire environment

Works everywhere Aura runs.
